# Final Presentation (Assessment)

## Overview

This is the final assessment for the Weather Intelligence Pipeline project. You will present your complete project, demonstrate your end-to-end pipeline, and submit your final repository. The assessment evaluates everything you have built across Days 1–8.

## Presentation

Prepare and deliver a **10-minute presentation** covering the full project journey.

### Presentation Structure

#### 1. Introduction (1–2 minutes)
- Problem statement: What did you set out to predict and why?
- Cities and variables chosen
- What "Can we trust this data?" means for your project

#### 2. Data Pipeline (2–3 minutes)
- Architecture overview (show your pipeline diagram from Day 5)
- Data sources and ingestion approach
- Database design (show schema)
- **Live demo**: Run your pipeline (or show it running) to prove it works end-to-end

#### 3. Data Quality Findings (2 minutes)
- Key data quality issues discovered
- How you handled them
- Your overall trust assessment of the data

#### 4. Statistical Analysis (2–3 minutes)
- Most interesting EDA findings (show 2–3 key visualisations)
- Hypothesis testing and correlation results (focus on the most impactful)
- Feature selection highlights

#### 5. Prediction Model (2–3 minutes)
- Prediction target and feature set
- Model comparison results
- Best model performance with confidence intervals
- Show a prediction vs. actual plot

#### 6. Conclusions & Reflections (1–2 minutes)
- Key takeaways
- What worked well, what you would do differently
- Answer: "Can we trust this data?" — your final verdict
- What would you add with more time?

### Presentation Requirements

- **Format**: Slide deck (Google Slides, PowerPoint, or Jupyter Slides) + live demo
- **Duration**: 10 minutes maximum + 3–5 minutes Q&A
- **Visuals**: Include at least 5 figures/charts from your analysis (not screenshots of code)
- **Code**: Show code only briefly when explaining the pipeline architecture — the audience cares about results, not syntax
- **Demo**: At minimum, show the pipeline running and a prediction being made

## Final Repository Requirements

Your submitted repository must contain:

- **Complete, well-organised codebase** in `src/` with all pipeline modules
- **All 8 daily notebooks** (day_01 through day_08), clean and runnable top-to-bottom
- **`README.md`** updated with final project description, setup instructions, and results summary
- **`reports/`** folder with key figures and the data quality report
- **`requirements.txt`** complete and tested

## Submission

Ensure all code is committed and the repository is clean. Submit the link to your final Pull Request.
